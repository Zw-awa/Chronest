# 安全策略

中文 | [English](SECURITY.EN.md)

## 支持范围

Chronest 当前处于 `pre-1.0` 阶段。在正式版本策略发布前，只能默认最新主分支状态为受支持版本。

## 漏洞报告方式

以下问题不要直接公开提 issue：

- 任意命令执行路径
- 插件权限绕过
- 密钥泄漏
- 不安全的 Git 自动化行为
- 路径穿越或文件系统越界

请先私下联系维护者，并尽量提供：

- 受影响的 commit 或分支
- 复现步骤
- 影响范围
- 建议缓解方式

## 处理目标

- 确认收到报告
- 复现并评估影响
- 准备修复或缓解措施
- 在合适时机进行协调披露
