# 贡献指南

中文 | [English](CONTRIBUTING.EN.md)

## 适合贡献的方向

Chronest 当前更欢迎以下方向的贡献：

- 日历驱动的任务调度体验
- 本地运行时稳定性
- 插件模型与权限边界
- Git 官方插件能力
- 文档、测试、打包与可访问性

## 基本流程

1. 大改动请先开 issue 讨论。
2. Pull request 尽量保持单一目标。
3. 优先提交可审阅、可回滚的小步修改。
4. UI 变更请附截图。
5. 行为变化请说明测试方式。

## 本地开发

```bash
npm install
npm run tauri dev
```

常用验证命令：

```bash
npm run build
cargo check
```

## 代码要求

- 保持本地优先，不要无意中把核心能力做成强依赖云端。
- 不要静默扩大插件或窗口权限。
- 不要把密钥明文写入 SQLite。
- 任务执行行为要尽量显式、可预期。
- 除非明确在做视觉重构，否则请保持现有界面方向一致。

## 提交前检查

- 变更范围清晰，说明明确
- 用户可见行为已写入文档
- 构建通过
- UI 变更附截图
- 权限、安全或插件边界变化已说明
