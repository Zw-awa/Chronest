# Security Policy

[中文](SECURITY.md) | English

## Supported versions

Chronest is pre-1.0. Until a stable release policy is published, only the latest default branch state should be considered supported.

## Reporting a vulnerability

Do not open public issues for security vulnerabilities involving:

- Arbitrary command execution paths
- Plugin permission bypass
- Secret storage leaks
- Unsafe Git automation behavior
- Path traversal or filesystem escape issues

Instead, report privately to the maintainers first. Include:

- Affected commit or branch
- Reproduction steps
- Impact assessment
- Any suggested mitigation

## Response goals

- Acknowledge receipt
- Reproduce and assess impact
- Prepare a fix or mitigation
- Publish a coordinated disclosure when appropriate
